# 画面キャプチャ → PDF

デスクトップ画面の指定した範囲を連続スクリーンショットし、矢印キーで対象アプリのページを自動でめくりながら全ページを取得、最後に 1 つの PDF に結合する Windows 向けツールです。

- 対象: 矢印キーでページ送りできるビューアー全般（PDF ビューアー、PowerPoint、電子書籍リーダー等）
- **完全ローカル動作**。ネットワーク通信・テレメトリ・外部送信は一切ありません
- 透かし・ロゴ・課金・機能制限なし

## 動作環境

| 項目 | 内容 |
|---|---|
| OS | Windows 10 / 11（64bit） |
| Python | exe 版なら不要。ソースから動かす場合のみ 3.9 以上（3.11 推奨） |

## 起動する

### 方法 1: exe を使う（Python 不要）

`BookToPDF.exe` をダブルクリックするだけです。インストール作業も管理者権限もいりません。同梱の `使い方.pdf` に、下の「使い方」以降と同じ内容が入っています。

初回起動だけ数秒かかります。単一ファイル形式のため、起動時に一時フォルダへ展開するからです。

exe 自体を作る手順は「開発者向け」を参照してください。

### 方法 2: ソースから動かす

```powershell
# 1. このリポジトリを取得して移動
git clone <このリポジトリのURL>
cd BookToPDF

# 2. 仮想環境を作成して有効化
python -m venv .venv
.venv\Scripts\activate

# 3. 依存パッケージをインストール
pip install -r requirements.txt

# 4. 起動
python main.py
```

2 回目以降は `.venv\Scripts\activate` → `python main.py` だけで起動できます。

## 使い方

画面の上から順に操作するだけです。

1. **キャプチャ範囲を選択** — ボタンを押すと画面全体が薄暗くなるので、取り込みたい部分（ビューアーのページ表示部分）をマウスでドラッグして囲みます。Esc でキャンセルできます。
   - **「画面全体」ボタン**を押せば、ドラッグせずに画面全体を範囲にできます。この場合はビューアーを最大化しておくと、本アプリのウィンドウが写り込みません。
2. **ページ送りを設定** — ページをめくる矢印キーを選びます（右送りのビューアーなら既定の「右 (→)」のまま）。数値はまず既定値のままで大丈夫です。
3. **出力を確認して「キャプチャ開始」** — カウントダウン（既定 4 秒）の間に、**取り込みたいビューアーをクリックして前面にしてください**。あとは自動でページをめくりながら撮影し、最終ページを検知すると止まって保存します。

### 保存のしかた

実行のたびに、保存先フォルダ（既定: デスクトップ）の中に **`capture_日時` フォルダ**が作られ、その中に成果物が保存されます。上書きの心配はありません。完了時に「フォルダを開きますか？」と聞かれるので、そのまま結果を確認できます。

```
Desktop/
└── capture_20260714_1030/
    ├── capture.pdf      ← 全ページを 1 つに結合した PDF
    ├── page_0001.png    ← ページごとの画像（出力形式で PNG を含めた場合）
    ├── page_0002.png
    └── ...
```

**出力形式**は 3 種類から選べます:

| 出力形式 | 向いている用途 |
|---|---|
| PDF + 画像 (PNG)（既定） | とりあえず全部。閲覧も再利用もできる |
| PDFのみ | 読む・共有する・タブレットで見る |
| 画像 (PNG) のみ | スライドや資料への貼り付け、画像編集、後から OCR |

> ページごとのファイルを PDF ではなく PNG にしているのは、1 ページ単位の用途（貼り付け・編集・OCR）では画像ファイルの方が扱いやすく、1 ページだけの PDF は PNG に対する利点がないためです。まとめて読むなら結合 PDF の方が向いています。

### ページ送りの待ち時間は自動で調整されます

待ち時間を手で調整する必要はありません。本ツールはキーを送ったあと、**実際に画面が変わるまで待ちます**。

- 描画が重いページでも、変わるまで待つので**取りこぼしません**
- キーがビューアーに届かなかった場合は、**自動でキーを送り直します**（最大 2 回）
- 送り直しても画面が変わらないときだけ、末尾に到達したと判断して停止します

### 途中で止めたいとき

- **Esc キー**（どのウィンドウが前面でも効きます）または「停止」ボタン
- それまでに取得したページで PDF が作られます

### パラメータの目安

通常はすべて既定値のままで動きます。

| 項目 | 既定値 | 説明 |
|---|---|---|
| 最短待機 (ms) | 300 | キーを送ってから撮るまでの**最低**待ち時間。以降はページが変わるまで自動で待つので、基本は変更不要です（手動モードでは撮影間隔になります） |
| 最大ページ数 | 500 | 暴走防止の安全上限。ここに達すると自動で止まります |
| 重複検知しきい値 | 3 | ページが変わったかどうかの感度。**ページが飛ぶときは小さく**、**最終ページで止まらないときは大きく**します |
| 開始前の待機 (秒) | 4 | ビューアーを前面にするための猶予 |
| 出力形式 | PDF + 画像 (PNG) | 上の「保存のしかた」を参照 |
| PDF品質 | 高画質 | 高画質 = 無劣化 PNG 結合。標準 / 軽量は縮小 + JPEG 化でファイルが小さくなります |

各入力欄にマウスを乗せると説明がその場で表示されます。

## 重要な注意

- **キー送信は「そのとき前面にあるウィンドウ」に届きます。** カウントダウン中に必ず取り込みたいビューアーをクリックして前面にしてください。前面化を忘れると、別のアプリに矢印キーが送られます。
- **私的利用の範囲でお使いください。** 自分の資料・スライドなど、複製が認められたコンテンツの PDF 化を想定しています。取り込み対象アプリの利用規約と著作権法などの関連法令を必ず守ってください。DRM の解除や保護されたコンテンツへのアクセスを支援するものではありません。
- 撮影した画像は一時フォルダ（`%TEMP%`）で作業し、保存完了時に自動削除されます。PDF 生成に失敗した場合も、撮ったページは PNG として保存先フォルダに退避されるので失われません。
- Esc 監視とキー送信に pynput を使用しているため、セキュリティソフトによっては警告が出ることがあります。キー入力の記録・送信は一切行っていません（ソースコードで確認できます）。

## うまくいかないとき

| 症状 | 対処 |
|---|---|
| 選択した範囲と撮れた範囲がずれる | 表示スケール変更後はアプリを再起動してください。マルチモニタ環境ではプライマリモニタでの使用を推奨します |
| 「ページが変わりません。キーを送り直しています…」が出て途中で止まる | キーがビューアーに届いていません。**ビューアーが前面にあるか**、そのビューアーが**その矢印キーでページを送れるか**を確認してください（アプリによっては PageDown や別のキーが必要です）。ビューアー上を一度クリックしてから開始し直すと直ることがあります |
| 最終ページで止まらない | 「重複検知しきい値」を大きくする。アニメーションやカーソル点滅が範囲に入らないように選択し直すのも有効です |
| 似たページが続く資料で早く止まる | 「重複検知しきい値」を小さくする |
| 手動でめくりたい | ページ送りキーを「なし（手動でめくる）」にすると「最短待機」の間隔で撮り続けます。終わったら Esc で停止（このモードでは自動停止しません） |

## 開発者向け

### 配布物のビルド（exe + 使い方 PDF）

```powershell
.venv\Scripts\activate
python build.py
```

これだけで `dist\` に次の 2 つが生成されます。PyInstaller が未インストールなら自動で入るので、事前準備は不要です。

| 生成物 | 内容 |
|---|---|
| `dist\BookToPDF.exe` | Python 不要の単体実行ファイル（約 22 MB、コンソール非表示） |
| `dist\使い方.pdf` | 利用者向けマニュアル（A4 / 2 ページ） |

この 2 ファイルを渡すだけで、Python 環境のない Windows でそのまま使えます。

**使い方 PDF は、この README.md から生成されます。**「使い方」「重要な注意」「うまくいかないとき」の各セクションを抜き出して A4 に組版したものです（セットアップや開発者向けの節は利用者には不要なので除外されます）。使い方を書き換えたら `python build.py` を流し直せば、exe とマニュアルの内容が必ず揃います。

- 載せる範囲を変えたいときは `manual_pdf.py` の `INCLUDE_SECTIONS` を編集してください。**逆に言うと、README の `## 使い方` などの見出し名を変えるとその節が PDF から落ちます**。見出しを変えるときは `INCLUDE_SECTIONS` も合わせて直してください
- 組版は Pillow で A4 ページを描画し、本アプリ自身の `pdf_builder.build_pdf`（img2pdf）で結合しています。新たな依存パッケージは増えません。日本語フォントは Windows 標準の游ゴシック / メイリオ / MS ゴシックを使うため、PDF 生成は Windows 上でのみ動きます
- pynput でキー送信を行う都合上、セキュリティソフトが未署名の exe を警告することがあります。配布するならコード署名を検討してください
- exe の起動を速くしたい場合は、`BookToPDF.spec` の `EXE(...)` を `COLLECT` 形式（フォルダ配布）に変更してください

### テスト

```powershell
# ユニットテスト（GUI・実画面キャプチャは対象外の純粋ロジックのみ）
python -m pytest tests -v
```

実画面のキャプチャと GUI は自動テストの対象外なので、次の手順で手動確認します。

1. 複数ページの PDF などをビューアーで開く
2. 本アプリで表示部分を範囲選択 → 「キャプチャ開始」
3. カウントダウン中にビューアーをクリックして前面化
4. 最終ページで自動停止することを確認
5. 保存された PDF を開き、全ページが順番どおり入っていることを確認

### プロジェクト構成

```
├── main.py              # エントリポイント + GUI
├── region_selector.py   # キャプチャ範囲選択オーバーレイ
├── capture_engine.py    # キャプチャループ本体
├── pdf_builder.py       # 画像 → PDF 結合
├── build.py             # 配布物の一括ビルド（exe + 使い方 PDF）
├── manual_pdf.py        # README.md → 使い方 PDF の組版
├── BookToPDF.spec       # PyInstaller 設定
├── requirements.txt
└── tests/
    └── test_core.py     # 純粋ロジックのユニットテスト
```
